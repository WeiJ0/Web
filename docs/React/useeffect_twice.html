React 18 useEffect 執行兩次問題

前言​

這次參加了 The F2E 的活動，主題是做滾動視差的互動式網站，這次想說試試看 Vite + React 開發，結果在寫 ScrollTrigger 時候發現，怎麼樣都沒辦法照我寫的內容去執行，滾動範圍不對、固定視窗固定不起來，開始跟結束位置也不對。很胡鬧的在亂跑一通

一開始是以為我 gsap 的部分寫得有問題，從頭到尾把 gsap ScrollTrigger 的文件翻了一遍，發現沒有類似的問題，後來在論壇上才找到類似的問題，而大大認為是重複去做了 ScrollTrigger 的設定。

看到這邊我回頭去看 code，我寫在 useEffect下，並在最後有加上參數 [] 只執行一次，應該沒甚麼問題才對，
又繼續去 Google React 是不是跟 GSAP 不合，也沒有相關文章，最後就在萬念俱灰下在 useEffect 中下了一個 console.log('run') 結果瀏覽器吐給我

run
run

哇靠，還真的執行了兩次 !? 兩個字 大中計

解決方法​

於是趕快去查為什麼會這樣，還是我 React 其實重頭到尾都學錯了，後來才發現原來在 React 18 版本後 useEffect 在嚴格模式下的確會執行兩次

那麼最快的方法的確是 關閉嚴格模式。

關閉嚴格模式​

create-react-app or vite​

index.js
root.render(
  {/* <StrictMode> */}
    <App />
  /* </StrictMode> */
);

next.js​

next.config.js
{
  reactStrictMode: false;
}

對 useEffect 進行處理​

可能有些人會對於將嚴謹模式關閉抱有疑慮及覺得不妥，那另一個方式就是對 useEffect 下手了
這邊使用 useRef 及 useState 協助判斷是否已經執行過該次的 useEffect ，如果執行過會直接 return 中止涵式

import { useState, useEffect, useRef } from "react";

export const useEffectOnce = (effect) => {
  const effectFn = useRef(effect);
  const destroyFn = useRef();
  const effectCalled = useRef(false);
  const rendered = useRef(false);
  const [, refresh] = useState(0);

  if (effectCalled.current) {
    rendered.current = true;
  }

  useEffect(() => {
    if (!effectCalled.current) {
      destroyFn.current = effectFn.current();
      effectCalled.current = true;
    }

    refresh(1);

    return () => {
      if (rendered.current === false) return;
      if (destroyFn.current) destroyFn.current();
    };
  }, []);
};

而在將原本 ScrollTrigger 的程式放進去 useEffectOnce 果真就正常了
