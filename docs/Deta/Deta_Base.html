Deta Base 微型 NoSQL

前言​

因為目前用到的都是 Node.js 居多，所以文章會以 Node 為主 ，python 可能要自行觀看官方文件
前置作業會需要到 Deta 註冊會員，會需要後台的一些 Token 或是 Key，測試上可以使用 Postman 較為方便

如果要使用 Node.js + Express.js 架設伺服器的話可以從 安裝 SDK 開始往下看，若只要使用 API 功能請從 HTML API 操作閱讀

安裝 SDK​

npm install deta
or
yarn add deta

取得 project key​

於後台左方列表最下面選擇 project keys，然後按下 Create Key

caution

project key 僅會出現這一次，所以需要把 key 存下來，關掉就只能重新產生了

建立資料集 & 新增一筆資料​

index.js
const { Deta } = require("deta");
const deta = Deta("[project key]"); // [project key] 需於後台新增
const db = deta.Base("todos"); // [DB Name] 可自定義名稱

const addTodo = async () => {
  // deta_base 的操作都需要使用 await 所以需要定義 async function
  await db.put({
    // 使用 put({內容}) 方式新增，後續會有詳細介紹
    content: "學習 Deta Base",
    isFinished: false,
  });
};
addTodo();
cmd
node index

執行後不會有什麼訊息，不過到後台重新整理就發現，Bases下多出剛剛定義的 todos，在點擊 todos 就可以看到資料
除了我們輸入的 content 和 isFinished 外，還有系統產生的 key 作為唯一值(ID)

資料庫操作方法​

新增資料 put & putMany & insert​

deta 提供兩種方法來操作新增資料，兩者的差別是 put 能夠快速的存取資料庫，
若 key 沒有重複就會新增資料，若 key 已存在就會覆蓋資料 (修改用)。

insert 則是新增資料，不過若遇到 key 重複則會引發錯誤，且其效能也是需要 put 的兩倍時間
所以官方推薦使用 put

await db.put(data, key, option);
await db.insert(data, key, options);
  • data [必填] 傳入的資料，可以是 object 、string 、boolean 、array
  • key 唯一值，可以自行指定，為 string 型態，若空白則由系統自動產生
  • option 選項 [object]
    • expireIn 於新增後幾秒鐘過期
    • expireAt 於指定日期過期
await db.put({ content: "學習 Deta Base", isFinished: false }, "", {
  expireIn: 10,
});
// 若有 option 選項又交由系統產生 key 值，則需要帶入空字串於 key value

可以看到設置過期秒數後後台會出現過期時間戳的欄位，而 10 秒後重新 Fetch 該筆就被刪除了

而除了 put 外，還有一個 putMany 可以一次新增或修改多筆的資料進資料庫，需要使用陣列操作

await db.put([
  { content: "學習 Deta Base", isFinished: false },
  { content: "寫 Deta Base 教學文件", isFinished: false },
]);

取得資料 get or fetch​

deta 提供兩種方法操作取得資料，get 需要填入 key 來取得特定值
而 fetch 則是可以更彈性的下搜尋條件，若無條件則是返回全部值

await db.get(key);
  • key [必填] 空會返回錯誤 : Key is empty
await db.get('izwrljlo2h1p');

//response
{ content: '學習 Deta Base', isFinished: false, key: 'izwrljlo2h1p' }
await db.fetch(query, options);
  • query 查詢條件，若空則回傳所有資料，但有 1MB 的限制，所以超過限制就要執行多次查詢，
    並使用 last 項目紀錄上次找到哪裡
  • options 選項 [object]
    • limit 查詢項目的限制值，最小值為 1
    • last 回傳上一次查詢最後一個項目，讓第二次查詢從這個項目往後找
await db.fetch('key': 'izwrljlo2h1p')

//response
{
   items: [
    { content: '學習 Deta Base', isFinished: false, key: 'izwrljlo2h1p' }
   ],
   count: 1,
   last: undefined
}
// 因為 fetch 可以回傳多筆資料，所以會被放在 items 陣列中，系統也貼心地回傳筆數，及上次找到哪裡

官方的範例去 fetch 所有資料，但資料大於 1mb

let res = await db.fetch();
let allItems = res.items;

// 如果前一次查詢 last 回傳還有資料代表需要進行再次查詢，而再次查詢就從 last 項目後開始找
while (res.last) {
  // 使用 while 直到 last 回傳 undefined
  res = await db.fetch({}, { last: res.last });
  allItems = allItems.concat(res.items);
}

更新資料 put & update​

前面有提到 put 如果遇到 key 值相同將覆蓋該資料，可以做到更新資料的功能，另外則是 update

await db.update(updates, key, options);
  • updates [必填]，要修改的內容必須使用陣列包裝
  • key [必填]，要修改的 key 值
  • options 如前面 put options
  • update 專屬的操作方法
    • Increment 對於原資料數字的增加 base.util.increment(數字)
    • Append 對於原資料陣列項目的新增(新增在後) base.util.append(內容)
    • Prepend 對於原資料陣列項目的新增(新增在前) base.util.prepend(內容)
    • Trim 移除原資料的該欄位 base.util.trim()

這邊用官方的舉例說明：

deta.Base('users')
{
  "key": "user-a",
  "username": "jimmy",
  "profile": {
    "age": 32,
    "active": false,
    "hometown": "pittsburgh"
  },
  "on_mobile": true,
  "likes": ["anime"],
  "purchases": 1
}
const updates = {
  "profile.age": 33, // 若原資料有該欄位則更新
  "profile.active": true, // 若原資料有該欄位則更新
  "profile.email": "jimmy@deta.sh", // 若原資料無欄位則新增該欄位
  "profile.hometown": users.util.trim(), // 移除 profile.hometown 欄位
  "on_mobile": users.util.trim(), // // 移除 on_mobile 欄位
  "purchases": users.util.increment(2), // 將原本 purchases + 2
  "likes": users.util.append("ramen") // 將原本 likes.push('ramen')
}
const res = await db.update(updates, "user-a");

//response
{
  "key": "user-a",
  "username": "jimmy",
  "profile": {
    "age": 33,
    "active": true,
    "email": "jimmy@deta.sh"
  },
  "likes": ["anime", "ramen"],
  "purchases": 3
}

刪除資料 delete​

await db.delete('izwrljlo2h1p');

HTML API 操作​

URL 網址​

https://database.deta.sh/v1/{project_id}/{base_name}

project_id 位於後台的 Project Keys 頁面右上方，base_name 則為資料庫名稱

Header 設置​

  • AUTH ：存取 API 需要使用授權驗證 X-API-KEY: [project key]
  • Content-Type: application/json

project key 取得請參閱 取得 project key 

fetch("URL", {
  headers: {
    "X-API-Key": "project key",
    "Content-type": "application/json",
  },
});

API 資料庫操作​

基本上函數與使用 Node 來執行相同。

新增資料 put & insert​

-PUT /items

// request
{
   "items": [
        {
            "key": {key}, //未填則系統產生
            "content": "學習 deta",
            "isfinished": true
        },
        {
            "key": {key}, //未填則系統產生
            "content": "寫 deta 教學",
            "isfinished": false
        }
    ]
}

// response 207
{
    "processed": {
        "items": [
           // 成功項目
        ]
    },
    "failed": {
       "items": [
           // 失敗項目
       ]
    }
}
caution

可能失敗原因如下

  • key 非字串，如 object,array,boolean , 兩個相同的 key
  • items 陣列最多項目為 25
  • 總請求大小超過限制 16 MB 或 items 單一項目超過 400KB

取得資料 get & query​

get 和 Node 操作一樣透過傳入 key 取得特定項目

-GET /items/{key}

//response
{
    "content": "學習 Deta Base",
    "isFinished": false,
    "key": "izwrljlo2h1p"
}

-POST /query

參數與 Node 操作相同，可參考 資料庫操作方法 - 取得資料-get or fetch

//request
{
   "query": [
        {"user.hometown": "Berlin", "user.active": true},
        {"user.age?lt": 40}
   ],
   "limit": 5,
   "last": "afsefasd"
}

更新資料 put & update​

參數與 Node 操作相同，可參考 資料庫操作方法 - 取得資料-get or fetch

{
   "set" : { // 賦予新值放在 set object 下方
     "profile.age": 33,
     "profile.active": true,
     "profile.email": "jimmy@deta.sh"
   },

   "increment" :{
     "purchases": 2
   },

   "append": {
     "likes": ["ramen"]
   },
   "delete": ["profile.hometown", "on_mobile"] // trim 改為 delete
}
