Deta Micros 部屬 Express 程式

前言​

在練習 Node Express.js 時，上網搜尋一定會有上架到 Heroku 的教學，透過免費的服務可以獲得一個線上的網址 run 你的伺服器程式， 但隨著 Heroku 免費服務即將下架時，Deta Micros 就可以成為新的練習服務啦~

前置作業需要安裝 Node NPM Express 及 Deta cli，前面三個就不多做說明，從 cli 開始吧

安裝 Deta cli​

  • windows powershell
  • mac & linux cmd
iwr https://get.deta.dev/cli.ps1 -useb | iex
curl -fsSL https://get.deta.dev/cli.sh | sh

安裝結束會出現

Deta was installed successfully to C:\Users\user\.deta\bin\deta.exe
Run 'deta --help' to get started

這時候就可以輸入 deta --help 進行測試，若有出現 deta 的說明文字即成功

Deta command line interface for managing deta micros.
Complete documentation available at https://docs.deta.sh

Usage:
  deta [flags]
  deta [command]

Available Commands:
  auth        Change auth settings for a deta micro
  clone       Clone a deta micro
  cron        Change cron settings for a deta micro
  deploy      Deploy a deta micro
  version     Print deta version
  visor       Change visor settings for a deta micro
  watch       Deploy changes in real time

Flags:
  -h, --help   help for deta

Use "deta [command] --help" for more information about a command.

登入 Deta​

請先至 Deta 註冊會員 並登入在執行下列步驟較為順暢

date login

首先若為 windows 會先跳出防火牆的提示請你協助開啟，並且會由瀏覽器傳送已登入的資料給 cli

> deta login
Please, log in from the web page. Waiting...
https://web.deta.sh/cli/49755
Logged in successfully.

建立新專案​

可以先到 deta 的後台建立一個專案，+ New project

Region 為伺服器的所在區域，選擇越近的速度會越快，這邊會建議選擇 Asia (Singapore)

deta new --node [伺服器名稱] --project [project 名稱]
tip

如果使用 python 則第一個參數為 --python ，但這篇以 node 為主，再麻煩參閱官方文件

建立成功則會回傳 micro 的詳細資訊

{
    "name": [伺服器名稱],
    "id": [伺服器 id],
    "project": [專案 id],
    "runtime": "nodejs14.x", //程式版本
    "endpoint": [前台網址],
    "region": [伺服器地區]
    "visor": [伺服器 Log功能],
    "http_auth": [前台是否需要驗證]
}

同時後台也會出現相關資訊

部屬一個 Express 程式​

在創建 micro 後，目錄下會新增一個伺服器名稱的資料夾，這時候 powershell 需要進入該資料夾執行 npm 安裝

npm init -y
npm install express

再新增 index.js 檔案，完成後執行 deta deploy

index.js
const express = require("express");

const app = express();

app.get("/", async (req, res) => {
  res.send("Hello World");
});

module.exports = app;
> deta deploy

Deploying...
Successfully deployed changes
Updating dependencies...

+ express@4.18.1
added 57 packages from 42 contributors and audited 57 packages in 4.79s

6 packages are looking for funding
  run `npm fund` for details

found 0 vulnerabilities

這時候打開剛剛 powershell create 後回傳的 endpoint 即會出現 Hello World! 即完成。

info

打開前台網址應該都可以看到 Hello World!，但若是出現下列錯誤則表示可能是你的瀏覽器禁用了 cookie 驗證 因為預設的 http_auth 都是 disabled 需驗證，這時候就要透過 cli 將前台轉為不用驗證(開放狀態)

{
    "errors":["Unauthorized"]
}

cli 輸入

> deta auth disable

Successfully disabled http auth
