React Hook-Form 送出事件 VS 多 useState

August 9, 2022 · 3 min read
WeiJie

前言​

最近在學習 React Hook 中有發現，在製作註冊表單這種多 Input 的頁面時

往往就需要宣告許多變數和 useState()

const [account, setAccount] = useState('')
const [password, setPassword] = useState('')
const [confirmPassword, setConfirmPassword] = useState('')
const [name, setName] = useState('')
const [email, setEmail] = useState('')
const [phone, setPhone] = useState('')

return {(
    <>
        <input value={account} placeholder="account"
            onChange={e => setAccount(e.target.value)} />
        <input value={password} placeholder="password"
            onChange={e => setPassword(e.target.value)} />
        <input value={confirmPassword} placeholder="password again"
            onChange={e => setConfirmPassword(e.target.value)} />
    </>
)}
...

如果又有地址、載具、愛心碼等等電商用的欄位，豈不是光這些東西就宣告了一堆變數
寫了一堆 onChange 事件使用不同的 methods 去修改值
當有了這些想法之後就開始再想有甚麼方法可以簡化，再逛 DEV 的時候就發現了這篇
