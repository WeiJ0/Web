使用 Deta 服務打造一支 TodoList API

September 10, 2022 · 9 min read
WeiJie

前言​

近期因為 Heroku 養套殺 免費方案即將下架的緣故，暫時沒有一個可以免費部屬 node 相關程式的地方，所以爬文後找到了 Deta，發現除了限制多了一點之外，拿來練習還是擋得住的。

所以再學習的第一個目標，就是用 Deta 建立一個 Todos 的 API 並部屬上去 Deta

環境建置​

環境建置的部分需要安裝 Deta 及建立一個 Deta Base 和 Deta Micros ，在筆記區已各寫完一份教學可以參閱著執行

建立 Express.js 環境​

若參照著 Deta Micros 文章最後一個已經成功建立一個 Deta 網站
並可以成功看到 Hello World 了。 這時候就要來寫 Express Todos API 的內容，先提供需要建置的架構

  • 取得 Todos 列表
  • 新增 Todo
  • 修改 Todo 狀態 (未完成/完成)
  • 刪除 Todo

那在建立以上的 API 前，需要做一些 Express 的前置作業，讓伺服器環境能夠順利被 Request

npm install cors

cors，用來解決 Request 的 cors 存取問題

index.js
const express = require("express");
const cors = require("cors");

const { Deta } = require("deta");
const deta = Deta(["PROJECT_KEY"]);
const db = deta.Base("todos");

const app = express();
app.use(express.json());
app.use(cors());

建置 Todo API​

取得所有 Todo​

在 Deta Base 微型 NoSQL - 資料庫操作方法 取得資料 get or fetch 有提到，若要取得不只一筆的資料時得使用 fetch
而因為 fetch 有單次存取大小 1MB 限制，所以需要用迴圈的方式去結合資料並回傳，所以這邊寫了一個函式

  • queryObject 為篩選的條件，若沒有傳入就是空陣列，代表取得所有資料的意思
const fetchDBData = async (queryObject = {}) => {
  let list = await db.fetch(queryObject);
  let allItems = await list.items;

  while (list.last) {
    list = await db.fetch(queryObject, { last: list.last });
    allItems = allItems.concat(list.items);
  }

  return allItems;
};

若以 GET 的方式存取 /todos/ 則回傳所有 Todo，而在回傳的格式也多加了一個 success 的 boolean 讓前端更方便判斷是否成功

  • /todos/
index.js
app.get("/todos/", async (req, res) => {
  res.json({ success: true, message: await fetchDBData() });
});

新增一筆 Todo 資料​

參考 Deta Base 微型 NoSQL - 資料庫操作方法 取得資料 put & putMany &insert ，因為效能及不會返回奇怪錯誤所以使用了 put 的功能

這邊有多做了一些處理如下：

  • /todos/add，參數為 content [string] 內容 及 inFinished [boolean] 是否完成
  • 避免塞入空值於資料庫中，所以判斷了內容欄位是否有傳入。
  • 若是否完成這個欄位沒有傳入值，則預設是未完成。
  • 使用了 try catch 避免於 deta-base 操作上發生預料外的錯誤
index.js
app.post("/todos/add", async (req, res) => {
  const { content, isFinished = false } = req.body;

  if (!content) {
    res.status(200).json({ success: false, message: "請輸入內容" });
    return;
  }

  try {
    const result = await db.put({ content, isFinished });
    res.status(201).json({ success: true, message: result });
  } catch (ex) {
    await res.json({ success: false, message: ex });
  }
});

​

切換 Todo 完成狀態​

參考 Deta Base 微型 NoSQL - 資料庫操作方法 更新資料 put & update 使用 update 完成

  • /todos/:id
  • 需判斷是否傳入 id 也就是 key 值，若沒有的話 update 會發生錯誤，所以要預先判斷擋掉
  • 和上面一點相同，若傳入了 key 值，但資料庫裏面沒有該筆 key 值一樣會 update錯誤，所以要先看存不存在
index.js
app.patch("/todos/:id", async (req, res) => {
  const { id } = req.params;

  if (!id) {
    await res.json({ success: false, message: "需要輸入 id" });
    return;
  }

  const item = await db.get(id);

  if (!item) {
    await res.json({ success: false, message: "查無該ID資料" });
    return;
  }

  try {
    const update = { isFinished: !item.isFinished };
    const result = await db.update(update, id);
    await res.json({ success: true, message: result });
  } catch (ex) {
    await res.json({ success: false, message: ex });
  }
});

刪除 Todo​

參考 Deta Base 微型 NoSQL - 資料庫操作方法 刪除資料 delete 使用 delete 完成

  • /todos/:id
  • 一樣判斷是否傳入 id，若沒有的話預先判斷擋掉
index.js
app.delete("/todos/:id", async (req, res) => {
  const { id } = req.params;

  if (!id) {
    await res.json({ success: false, message: "需要輸入 id" });
    return;
  }

  try {
    const result = await db.delete(id);
    await res.json({ success: true, message: result });
  } catch (ex) {
    await res.json({ success: false, message: ex });
  }
});

​

刪除已完成的 Todo​

同場加映，因為會使用之前於六角學院 React 直播班建立的 TodoList 作業上，裡面有一個功能就是刪除已完成的項目，原本的 API 是沒有該功能的， 所以需要用迴圈去執行 API，不過現在自己開 API 了就可以建置一支這樣的功能，前台就只要 request 一次就好。

先取得已完成的所有項目，一樣使用迴圈進行刪除

index.js
app.delete("/todos/all", async (req, res) => {
  try {
    const items = await fetchDBData({ isFinished: true });
    items
      .map((item) => item.key)
      .forEach(async (key) => {
        await db.delete(key);
      });
    await res.json({ success: true, message: "" });
  } catch (ex) {
    await res.json({ success: false, message: ex });
  }
});

開發模式與部屬模式​

在開發模式下是使用 Express 的 server 功能，在 localhost 環境下用 js fetch 或 postman 去測試的，所以程式碼要改為

index.js
app.listen(3032, () => {
  console.log("server is running");
});

module.exports = app;

並執行 

node index

等到正式要部屬上 deta 時再將 app.listen 這段註解，改用 module.exports = app;

index.js
//app.listen(3032, () => {
//  console.log("server is running");
//});
module.exports = app;

完整程式碼​

同步放置於 Github 上，可以直接 clone 執行

caution

僅需要注意一點是 DELETE /todos/all 需要放置在 DELETE /todos/:id 之前，否則 all 會被當作 id 傳入。

require("dotenv").config();
const express = require("express");
const bodyParser = require("body-parser");
const cors = require("cors");

const { Deta } = require("deta");
const deta = Deta(process.env.PROJECT_KEY);
const db = deta.Base("todos");

const app = express();

app.use(
  bodyParser.urlencoded({
    extended: true,
  })
);
app.use(bodyParser.json());
app.use(express.json());
app.use(cors());

app.get("/", (req, res) => res.send("Hello World!"));

const fetchDBData = async (queryObject = {}) => {
  let list = await db.fetch(queryObject);
  let allItems = await list.items;

  while (list.last) {
    list = await db.fetch(queryObject, { last: list.last });
    allItems = allItems.concat(list.items);
  }

  return allItems;
};

app.get("/todos/", async (req, res) => {
  res.json({ success: true, message: await fetchDBData() });
});

app.post("/todos/add", async (req, res) => {
  const { content, isFinished = false } = req.body;

  if (!content) {
    res.status(200).json({ success: false, message: "請輸入內容" });
    return;
  }

  try {
    const result = await db.put({ content, isFinished });
    res.status(201).json({ success: true, message: result });
  } catch (ex) {
    await res.json({ success: false, message: ex });
  }
});

app.patch("/todos/:id", async (req, res) => {
  const { id } = req.params;

  if (!id) {
    await res.json({ success: false, message: "需要輸入 id" });
    return;
  }

  const item = await db.get(id);

  if (!item) {
    await res.json({ success: false, message: "查無該ID資料" });
    return;
  }

  try {
    const update = { isFinished: !item.isFinished };
    const result = await db.update(update, id);
    await res.json({ success: true, message: result });
  } catch (ex) {
    await res.json({ success: false, message: ex });
  }
});

app.delete("/todos/all", async (req, res) => {
  try {
    const items = await fetchDBData({ isFinished: true });
    items
      .map((item) => item.key)
      .forEach(async (key) => {
        await db.delete(key);
      });
    await res.json({ success: true, message: "" });
  } catch (ex) {
    await res.json({ success: false, message: ex });
  }
});

app.delete("/todos/:id", async (req, res) => {
  const { id } = req.params;

  if (!id) {
    await res.json({ success: false, message: "需要輸入 id" });
    return;
  }

  try {
    const result = await db.delete(id);
    await res.json({ success: true, message: result });
  } catch (ex) {
    await res.json({ success: false, message: ex });
  }
});

/* 開發使用 */
/* app.listen(3032, () => {
  console.log("server is running");
}); */

module.exports = app;
