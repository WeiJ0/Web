切版​

切版 - 特殊的 checkbox

我的解法是用將原本的 checkbox 隱藏，再替換成從設計稿取出的 svg 或 png 檔
若是勾選的話就可以用 CSS 的 :checked 來選擇到替換成打勾勾圖，另外一張就是框框圖了
label 標籤包住 checkbox 及一個 span ，利用 label 的 for 功能也可以去觸發 checkbox 的切換

<label for="checkbox1">
  <input type="checkbox" id="checkbox1" />
  <span>原本 label 的文字</span>
</label>
label {
  display: flex;
  align-items: center;
  padding: 15px 0;
  width: 95%;
  line-height: 26px;
  border-bottom: 1px solid #e5e5e5;
  cursor: pointer;
}
input[type="checkbox"] {
  display: none;
}
input[type="checkbox"] + span {
  position: relative;
  padding-left: 36px;
  background: /* 框框圖片 */ no-repeat left center;
}
input[type="checkbox"]:checked + span {
  background: /* 勾勾圖片 */ no-repeat left center;
}

API​

API 及登入資訊實作

這次的有提供 API 可以做到註冊及登入，在 🔗文件 上有看到
既然要實作登入又有 Token 就必須把資料存起來，但這次沒有使用其他相關的套件管理
就改成使用 localstorage 來實作
先開一支隨便命名的 js 寫相關功能

common.js
// 將登入的資料存進去 user
export const setUserData = (data) => localStorage.setItem("user", JSON.stringify(data));
// 取得存入的 user
export const getUserData = () => JSON.parse(localStorage.getItem("user"));
// 取得 user authorization
export const getAuthToken = () => getUserData().authorization;
// 砍掉 user 資料
export const clearUserData = () => localStorage.clear();

完成後就要來接 API 啦，不過如何管理這些 API 們也是一門學問
如果每個 function 都去重新寫一次 Request 的話，萬一 API URL 或是需要傳遞的欄位名稱變了
豈不是要翻開專案一支一支抓來改，所以集中管理會方便許多
趕快回去找找 Mike 大的文章 🔗使用 Axios 你的 API 都怎麼管理？

tip

文章是使用 Axios 而我是用 fetch 達成，但大同小異啦
🔗Fetch 這篇寫的超詳細 - JavaScript Fetch API 使用教學 - OXXO.STUDIO

效仿著文章開了一支 API.js 來管理所有的 Request

API.js
import { getAuthToken } from "./common";
// API 的前綴網址
const API_URL = "https://todoo.5xcamp.us";

// 共用的 Fetch Option
let optionBase = {
  headers: {
    Accept: "application/json",
    "Content-Type": "application/json",
  },
};

// Users 功能
const userRequest = (method, path, data) => {
  let optionObject = Object.assign({}, { ...optionBase, method });

  // Fetch 在 GET 的情況下不能帶有 body 參數，所以先判斷是不是 POST 在加上該參數
  if (method === "POST") optionObject.body = JSON.stringify(data);

  return fetch(`${API_URL}/users/${path}`, optionObject);
};

// Todos 功能
const todoRequest = (method, path, data) => {
  let optionObject = Object.assign({}, { ...optionBase, method });

  // Todos 功能需要帶入 Authorization
  optionObject.headers.Authorization = getAuthToken();
  if (method === "POST") optionObject.body = JSON.stringify(data);

  return fetch(`${API_URL}/todos/${path}`, optionObject);
};

// User
export const apiLogin = ({ email, password }) =>
  // 預先定義好 method & path 這樣就只要傳參數就好了
  userRequest("POST", "sign_in", {
    user: {
      email,
      password,
    },
  });

實際使用上

login.jsx
import { apiLogin } from "../API";

apiLogin({ email, password })
  .then((res) => res.json())
  .then((resJson) => {
    // 處理
  })
  .catch((err) => {
    // 錯誤處理
  });

是不是簡潔許多呢，而若是大專案有許多地方會用到共同的 API
往後就只要修改 API.js 這支檔案就好，真香 👍👍

部屬 Github Page​

Router 配置

原本想說參照著🔗文件 就可以安然無恙的部屬上來
結果一打開 Github Page 出現一片空白，有了 Vue Cli 部屬的經驗，我想一定又是路由出問題了
後來參閱了 🔗stackoverflow - react-router-not-working-with-github-pages

有人提供了這方面的訊息指出 Github Page 不支援 react-router 預設的 BrowserRouter
需要改使用 HashRouter 來實作，更改程式碼如下：

main.js
const root = ReactDOM.createRoot(document.getElementById("root"));
root.render(
  <React.StrictMode>
    <HashRouter>
      <App />
    </HashRouter>
  </React.StrictMode>
);

// APP 內容
function App() {
  return (
    <Routes>
      <Route path="/" element={<Login />}></Route>
      <Route path="/SignUp" element={<SignUp />}></Route>
      <Route path="/Todo" element={<Todo />}></Route>
    </Routes>
  );
}

再重新部屬就成功有畫面了🎉 不過透過 HashRouter 的方式實現，網址上就會帶有 #
