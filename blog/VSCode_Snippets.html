設定自己的程式碼片段，讓開發 Turbo 起來 ⚡

October 28, 2022 · 3 min read
WeiJie

前言​

在 codeing 的過程常常會發現，好像有一個東西我超級常用
像是在寫 css 時，會有 mediaquery 定義不同裝置寬度的樣式

/* 平板 */
@media screen and (max-width: 1024px) {
}

/* 手機 */
@media screen and (max-width: 767px) {
}

雖然說現在已經有太多工具可以全域或更快的去寫這段判斷的程式碼，但在維護舊案時
總是有這麼幾個專案就是沒有使用這些預載入工具，也沒有 global 的 css
導致我要到這些網頁上一筆一筆的寫上 style，這時候程式碼片段工具就香了起來

設定方式​

首先在 vscode 的左下方有一個齒輪圖案，點下後可以看到 設定使用者程式碼片段 (User Snippets)

點下後可以看到上方會跳出所有的程式語言，這時候再去選擇要新增的語言，這邊就以 css 為例

開啟 css.json 檔後會看到上方有很多註解文字，主要都是在說格式是什麼，簡單整理後如下：

css.json
{
  "Snippet名稱": {
    "prefix": "在編輯器輸入的前綴字 (一個字串，多個為陣列)",
    "body": "帶出的程式內容 (單行字串，多行用陣列)",
    "description": "可不填，描述這個 Snippet 在做什麼"
  }
}

可以帶入 $1 $2 順序參數，此為 Tab 帶入程式碼後，會依序停在哪邊讓你輸入
例如我在 max-width: 插入 $1 那我按下，Tab 則會停在 px 讓我輸入是幾 px 而 $2 放在大括號中，按下第二次 Tab 即跳到輸入程式碼的地方

css.json
{
  "media max": {
    "prefix": "mediamax",
    "body": ["@media screen and (max-width:$1px){", "$2", "}"],
    "description": "media max-width"
  }
}

完成後儲存關掉 css.json 就可以到編輯器下測試了，若 Vscode 有登入的話也會同步到不同裝置
非常方便

網路工具​

而除了自己寫 json 外，網路上也有工具可以做編輯，還可以一次做到 vscode sublime atom 的配置檔 snippet generator

在左邊編輯自己的程式碼後將右邊的 json 再貼回使用者程式碼片段即可。
