解決 ReactRouter 回上頁後回到原本滾動的位置

August 19, 2022 · 4 min read
WeiJie

前言​

這幾天一樣是在做 六角React線上讀書會 的每週作業時，列表的部分並沒有做分頁的功能
而是使用 Lazy Loading 的作法往下展，而按下景點就會跑到景點的內容頁
但再回到列表時又要重頭滾過，這樣對使用者來說會很不方便

修改前​

作法​

在 JS 中取得及控制滾動位置是使用

// 取得目前網頁已滾動值
window.scrollY
// 將網頁滾動至某個地方
window.scrollTo({ top: /* 滾動的值 */ });

所以在這個專案中，需要去監聽滾動的值並存起來，等到回到列表頁時再讀出來
存起來的方法有很多種，若沒有載入相關管理的 plugin 的話
就可以使用 localstorage 來存，或是透過 Router 的 State 傳遞相關參數 這次則用後者來達成

存起來的部分
const [scrollV, setScrollV] = useState(0);

useEffect(() => {
  document.addEventListener("scroll", (e) => {
    setScrollV(window.scrollY);
  });
}, []);
傳遞參數的部分
// 透過 Router 切換到內頁後，會帶入 state 裡面一個 scroll 物件
<Link to={`/info/${Id}`} state={{ scroll }}></Link>;

// 內頁取得 state 需要使用到 useLocation
import { useLocation } from "react-router-dom";

export default function Detail() {
  const location = useLocation();
  // 取得 scroll 的數值，若沒有則為 0
  const scroll = location.state.scroll || 0;
}

其實這個問題應該使用 localstorage 比較方便，可以直接在列表頁存跟讀
若透過 state 則要先傳到內頁，再傳回來列表兩道手續
內頁的部分一樣把原本的 scroll 再透過 state 傳回列表頁

列表頁
import { useState, useEffect } from "react";
import { useLocation } from "react-router-dom";

export default function List() {
  const location = useLocation();
  let scroll = location.state ? location.state.scroll : 0;
  const [scrollV, setScrollV] = useState(0);

  useEffect(() => {
    // 取得景點資料渲染成列表
    renderData();
    // 監聽滾動值
    document.addEventListener("scroll", (e) => {
      setScrollV(window.scrollY);
    });
    // 滾動到儲存的值
    window.scrollTo({ top: scroll });
  }, []);
}

原本以為這樣就完成了，結果回來還是沒效果...
但去 console.log(scroll) 或是手動執行 window.scrollTo({ top: scroll }) 都沒問題
後來才想到可能執行的時間是景點還沒渲染完成前，所以滾到儲存的值是沒有反應的 儲存的值大於目前頁面的高度
所以只要讓他慢一點執行就可以了

setTimeout(() => {
  window.scrollTo({ top: scroll });
}, 100);
